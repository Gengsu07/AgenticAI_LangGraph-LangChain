# 🧠 Conversational RAG dengan Memory — Panduan Lengkap

## 📋 Apa Ini?
Teknik menambahkan **memory/chat history** ke RAG pipeline sehingga sistem bisa memahami
follow-up questions dan menjawab berdasarkan konteks percakapan sebelumnya.

---

## ❌ Masalah: RAG Tanpa Memory

RAG biasa hanya melihat **pertanyaan terbaru** tanpa konteks:

```
User: "Apa itu perawatan saluran akar?"  → ✅ Bisa dijawab
User: "Apa manfaatnya?"                 → ❌ GAGAL ("manfaat apa?")
User: "Berapa biayanya?"                → ❌ GAGAL ("biaya apa?")
```

**Penyebab:** Retriever hanya search berdasarkan query terakhir.
"Apa manfaatnya?" → mencari "manfaat" secara umum, bukan "manfaat perawatan saluran akar".

---

## ✅ Solusi: Conversational RAG

### Arsitektur:

```
User Question + Chat History
        │
        ▼
┌───────────────────────┐
│ History-Aware          │  ← Reformulasi: "manfaatnya" → "manfaat perawatan saluran akar"
│ Retriever              │
└───────────┬───────────┘
            │
            ▼
┌───────────────────────┐
│ Vector Store Retriever │  ← Search dengan query yang sudah jelas
└───────────┬───────────┘
            │
            ▼
┌───────────────────────┐
│ QA Chain               │  ← Jawab berdasarkan docs + chat history
└───────────┬───────────┘
            │
            ▼
      Answer + Save History
```

---

## 🔑 Komponen Kunci

### 1. `create_history_aware_retriever`
**Fungsi:** Reformulasi pertanyaan user berdasarkan chat history sebelum retrieve

```python
from langchain.chains import create_history_aware_retriever

contextualize_q_prompt = ChatPromptTemplate.from_messages([
    ("system", "Reformulate the question to be standalone..."),
    MessagesPlaceholder("chat_history"),
    ("human", "{input}"),
])

history_aware_retriever = create_history_aware_retriever(
    llm=model,
    retriever=retriever,
    prompt=contextualize_q_prompt
)
```

**Cara kerja:**
- Jika ada `chat_history` → reformulasi pertanyaan dulu, baru retrieve
- Jika kosong → langsung retrieve tanpa reformulasi

### 2. `create_stuff_documents_chain`
**Fungsi:** Menggabungkan dokumen yang di-retrieve dan menjawab pertanyaan

```python
from langchain.chains.combine_documents import create_stuff_documents_chain

qa_prompt = ChatPromptTemplate.from_messages([
    ("system", "Jawab berdasarkan konteks:\n{context}"),
    MessagesPlaceholder("chat_history"),
    ("human", "{input}"),
])

question_answer_chain = create_stuff_documents_chain(llm=model, prompt=qa_prompt)
```

### 3. `create_retrieval_chain`
**Fungsi:** Menggabungkan retriever + QA chain jadi satu pipeline

```python
from langchain.chains import create_retrieval_chain

rag_chain = create_retrieval_chain(
    retriever=history_aware_retriever,
    combine_docs_chain=question_answer_chain
)
```

**Output:** `{"answer": "...", "context": [docs], "input": "..."}`

### 4. `RunnableWithMessageHistory`
**Fungsi:** Auto load/save chat history per session

```python
from langchain_core.runnables.history import RunnableWithMessageHistory

conversational_rag = RunnableWithMessageHistory(
    rag_chain,
    get_session_history,
    input_messages_key="input",         # key pertanyaan user
    history_messages_key="chat_history", # key inject history
    output_messages_key="answer",        # key response yang disimpan
)
```

### 5. `SQLChatMessageHistory`
**Fungsi:** Storage backend untuk history

```python
from langchain_community.chat_message_histories import SQLChatMessageHistory

def get_session_history(session_id: str) -> SQLChatMessageHistory:
    return SQLChatMessageHistory(
        session_id=session_id,
        connection_string="sqlite:///conversation_history.db",
    )
```

---

## 🧪 Cara Pakai

### Invoke dengan Session ID:
```python
config = {"configurable": {"session_id": "user1_session1"}}

# Turn 1
r1 = conversational_rag.invoke({"input": "Apa itu veneer?"}, config=config)

# Turn 2 — Follow-up (memory otomatis!)
r2 = conversational_rag.invoke({"input": "Ada berapa jenisnya?"}, config=config)
```

### Session Isolation:
```python
# User 1
config1 = {"configurable": {"session_id": "user1"}}

# User 2 — history terpisah!
config2 = {"configurable": {"session_id": "user2"}}
```

---

## 📐 Perbedaan RAG Biasa vs Conversational RAG

| Aspek | RAG Biasa | Conversational RAG |
|-------|-----------|-------------------|
| Follow-up questions | ❌ Tidak bisa | ✅ Bisa |
| Memory | Tidak ada | Per session (SQLite/DB) |
| Retriever | Langsung search | Reformulasi dulu → search |
| Prompt | Hanya `{context} + {question}` | `{context} + {chat_history} + {input}` |
| Components | `retriever \| prompt \| model` | `history_aware_retriever + stuff_chain + RunnableWithMessageHistory` |

---

## ⚠️ Catatan Penting

1. **`MessagesPlaceholder("chat_history")`** — Wajib ada di kedua prompt (contextualize + QA)
2. **`input_messages_key`** harus match dengan key di prompt `{input}`
3. **`history_messages_key`** harus match dengan `MessagesPlaceholder` variable name
4. **`output_messages_key`** menentukan response mana yang disimpan ke history
