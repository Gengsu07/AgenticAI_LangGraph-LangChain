Catatan ini menjelaskan langkah demi langkah proses yang berjalan pada notebook [1.1.2-Simpleapp.ipynb](file:///d:/PROJECTS/AgenticAI_LangGraph-LangChain/Lecturer/Section%206-Get%20Started/1.1-openai/1.1.2-Simpleapp.ipynb). Tujuannya adalah membangun aplikasi **RAG (Retrieval-Augmented Generation)** sederhana yang bisa menjawab pertanyaan berdasarkan konten dari sebuah halaman web.

## Konsep Fundamental

Sebelum masuk ke kode, berikut adalah konsep kuncinya:
1.  **Loading**: Mengambil data dari sumber (Web, PDF, dll).
2.  **Splitting**: Memecah teks panjang menjadi potongan kecil (chunks) agar muat di memori LLM dan pencarian lebih akurat.
3.  **Embedding**: Mengubah teks menjadi deretan angka (vektor) agar komputer bisa memahami "makna" dan kemiripan antar teks.
4.  **Vector Store**: Database khusus untuk menyimpan vektor tersebut.
5.  **Retrieval**: Mencari potongan teks yang paling relevan dengan pertanyaan user.
6.  **Generation**: LLM (seperti GPT-4) menjawab pertanyaan dengan menggunakan informasi yang ditemukan tadi.

## Langkah 1: Persiapan Environment

Langkah pertama adalah menyiapkan "kunci" agar bisa menggunakan layanan OpenAI dan LangSmith (untuk tracking).

```python
import os
```

**Poin Penting:**

*   `LANGCHAIN_TRACING_V2`: Fitur opsional untuk memantau performa aplikasi kita di dashboard LangSmith.

## Langkah 2: Mencerna Data (Data Ingestion)

Kita mengambil konten dari sebuah website dokumentasi LangSmith.

**Kode:**
```python
from langchain_community.document_loaders import WebBaseLoader

loader = WebBaseLoader("https://docs.smith.langchain.com/tutorials/Administrators/manage_spend")
docs = loader.load()
```

**Penjelasan:**
*   `WebBaseLoader`: Alat bantu (tool) untuk "crawling" atau membaca isi halaman web.
*   Hasilnya (`docs`) adalah teks mentah dari seluruh halaman web tersebut.

---

## Langkah 3: Memecah Dokumen (Text Splitting)

Teks dari satu halaman web bisa sangat panjang. LLM memiliki batasan jumlah kata (context window). Kita perlu memecahnya.

**Kode:**
```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
documents = text_splitter.split_documents(docs)
```

**Penjelasan:**
*   `chunk_size=1000`: Setiap potongan maksimal 1000 karakter.
*   `chunk_overlap=200`: Ada irisan 200 karakter antar potongan. Tujuannya agar konteks kalimat tidak terputus tiba-tiba di tengah jalan.

---

## Langkah 4: Membuat Vector Store (Otak Pencarian)

Di sini kita mengubah teks menjadi angka (vektor) dan menyimpannya agar bisa dicari.

**Kode:**
```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import FAISS

embeddings = OpenAIEmbeddings()
vectorstoredb = FAISS.from_documents(documents, embeddings)
```

**Penjelasan:**
*   `OpenAIEmbeddings`: Model AI yang bertugas menerjemahkan teks menjadi vektor.
*   `FAISS`: Sebuah pustaka (library) dari Facebook untuk pencarian vektor yang sangat cepat. Ini bertindak sebagai database kita.

**Ilustrasi Konsep:**
> Teks: "Kucing makan ikan" -> Embedding -> [0.1, 0.5, 0.9]
> Ketika kita cari "Hewan lapar", sistem mencari vektor yang angkanya mirip dengan [0.1, 0.5, 0.9].

## Langkah 5: Menyiapkan LLM

Kita memanggil model otak utamanya, yaitu GPT-4o.

**Kode:**
```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")
```

---

## Langkah 6: Membuat Rantai Proses (Chain)

Ini adalah inti logika aplikasi. Kita menggabungkan **Prompt**, **LLM**, dan **Dokumen** yang ditemukan.

**Kode:**
```python
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("""
Answer the following question based only on the provided context:
<context>
{context}
</context>
""")

document_chain = create_stuff_documents_chain(llm, prompt)
```

**Penjelasan:**
*   PromptTemplate: Memberi instruksi ke AI, "Jawab pertanyaan HANYA berdasarkan konteks yang diberikan". Ini mencegah AI berhalusinasi atau menjawab ngawur.
*   `create_stuff_documents_chain`: Teknik sederhana dimana semua dokumen yang relevan "di-stuff" (dimasukkan) langsung ke dalam prompt.

---

## Langkah 7: Retrieval Chain (Menggabungkan Pencarian dan Jawaban)

Kita menghubungkan sistem pencarian (Vector Store) dengan sistem jawaban (Document Chain).

**Kode:**
```python
from langchain.chains import create_retrieval_chain

retriever = vectorstoredb.as_retriever()
retrieval_chain = create_retrieval_chain(retriever, document_chain)
```

**Alur Kerja `retrieval_chain`:**
1.  User input pertanyaan.
2.  `retriever` mencari di DB dokumen mana yang relevan.
3.  Dokumen relevan + Pertanyaan dikirim ke `document_chain`.
4.  LLM menjawab.


---

## Ringkasan Alur Data
1.  **Web** -> `Loader` -> **Documents**
2.  **Documents** -> `Splitter` -> **Chunks**
3.  **Chunks** + `Embeddings` -> **Vector Store (FAISS)**
4.  **Pertanyaan User** -> `Retriever` -> **Dokumen Relevan**
5.  **Dokumen Relevan** + **Pertanyaan** -> `LLM` -> **Jawaban**
