## рҹ“Ң Overview

Dokumen ini merangkum **3 konsep inti** dalam membangun aplikasi LLM dengan LangChain:

1. **Prompt Template** вҖ” Cara membuat template pesan yang reusable
2. **Output Parser** вҖ” Cara mengubah output LLM ke format yang diinginkan
3. **LCEL Chain** вҖ” Cara menyambungkan komponen menjadi pipeline menggunakan LangChain Expression Language

### Alur Kerja Lengkap

```
Input (dict) вҶ’ Prompt Template вҶ’ Model (LLM) вҶ’ Output Parser вҶ’ Output (string/list/json)
```

## 1пёҸвғЈ Membuat Prompt dengan `ChatPromptTemplate`

### Apa itu Prompt Template?

Prompt Template adalah **cetakan pesan** yang memiliki variabel placeholder (`{variabel}`). Kita bisa mengisi variabel ini dengan nilai berbeda setiap kali digunakan, tanpa menulis ulang seluruh prompt.

### Cara Membuat: `ChatPromptTemplate.from_messages()`

```python
from langchain_core.prompts import ChatPromptTemplate

# Definisikan template dengan placeholder {kota} dan {text}
Instruct_template = "You are cultural young Indonesian, please translate teks into bahasa gaul di {kota}"

# Buat template object menggunakan from_messages()
prompt = ChatPromptTemplate.from_messages([
    ("system", Instruct_template),   # System message (instruksi ke LLM)
    ("user", "{text}")               # User message (input dari pengguna)
])
```

### Cara Menggunakan: `.invoke()`

```python
# Isi variabel dan jalankan template
hasilprompt = prompt.invoke({
    "kota": "Jakarta Selatan",
    "text": "Hello, please introduce my name is Aksa. I live in Jaksel"
})

# Lihat hasil messages
hasilprompt.to_messages()
```

**Output:**
```python
[
    SystemMessage(content='You are cultural young Indonesian, please translate teks into bahasa gaul di Jakarta Selatan'),
    HumanMessage(content='Hello, please introduce my name is Aksa. I live in Jaksel')
]
```

### вҡ пёҸ Kesalahan Umum

| вқҢ Salah | вң… Benar |
| :--- | :--- |
| `ChatPromptTemplate.format_messages([...])` | `ChatPromptTemplate.from_messages([...])` |
| Menghasilkan `list` biasa, bukan template | Menghasilkan `ChatPromptTemplate` object |
| Tidak bisa `.invoke()` | Bisa `.invoke()` untuk isi variabel |

### Perbedaan Method pada ChatPromptTemplate

| Method | Fungsi | Return Type |
| :--- | :--- | :--- |
| `.from_messages([...])` | **Membuat** template baru (class method) | `ChatPromptTemplate` |
| `.invoke({...})` | **Mengisi** variabel dalam template | `ChatPromptValue` |
| `.to_messages()` | **Mengkonversi** hasil ke list of messages | `list[BaseMessage]` |

### Format Tuple pada Messages

```python
# Format: (role, content)
[
    ("system", "instruksi..."),    # вҶ’ SystemMessage
    ("user", "{variabel}"),       # вҶ’ HumanMessage
    ("assistant", "..."),         # вҶ’ AIMessage (opsional)
]
```

## 2пёҸвғЈ Membuat Messages Manual (Tanpa Template)

Selain menggunakan template, kita bisa membuat messages secara langsung:

```python
from langchain_core.messages import SystemMessage, HumanMessage

messages = [
    SystemMessage('You are expert explainer, please explain given concept step by step'),
    HumanMessage('How aircraft can fly?')
]

# Langsung invoke ke model
result = model.invoke(messages)
```

### Kapan Pakai Template vs Manual?

| Situasi | Gunakan |
| :--- | :--- |
| Prompt **tetap**, tanpa variabel | Messages manual (`SystemMessage`, `HumanMessage`) |
| Prompt **dinamis**, ada variabel `{...}` | `ChatPromptTemplate.from_messages()` |
| Perlu di-**chain** dengan komponen lain | `ChatPromptTemplate` (wajib) |

## 3пёҸвғЈ Output Parser

### Apa itu Output Parser?

Output Parser **mengubah format output** dari LLM. Secara default, LLM mengembalikan `AIMessage` object. Parser mengkonversi ini menjadi format yang lebih mudah diolah.

### `StrOutputParser` вҖ” Parse ke String

Mengambil isi `content` dari `AIMessage` dan mengembalikan **string biasa**.

```python
from langchain_core.output_parsers import StrOutputParser

parser = StrOutputParser()

# result = AIMessage object dari model.invoke()
text_output = parser.invoke(result)
# Output: string biasa tanpa wrapper AIMessage
```

**Sebelum Parser:**
```python
AIMessage(content='Aircraft fly because...', response_metadata={...})
```

**Sesudah Parser:**
```python
'Aircraft fly because...'
```

### `MarkdownListOutputParser` вҖ” Parse ke List

Mengambil semua **bullet points** dari output markdown dan mengembalikan **list of strings**.

```python
from langchain_core.output_parsers import MarkdownListOutputParser

md = MarkdownListOutputParser()
list_output = md.invoke(result)
# Output: ['item 1', 'item 2', 'item 3', ...]
```

**Output:**
```python
[
    '**Lift:** The upward force that counteracts gravity...',
    '**Weight:** The downward force due to gravity...',
    '**Thrust:** The forward force generated by engines...',
    '**Drag:** The backward resistance force...',
]
```

### Perbandingan Output Parser

| Parser | Import | Output | Kapan Digunakan |
| :--- | :--- | :--- | :--- |
| `StrOutputParser` | `langchain_core.output_parsers` | `str` | Butuh teks mentah saja |
| `MarkdownListOutputParser` | `langchain_core.output_parsers` | `list[str]` | Output berupa daftar/bullet points |
| `JsonOutputParser` | `langchain_core.output_parsers` | `dict` | Output berupa JSON |
| `PydanticOutputParser` | `langchain_core.output_parsers` | Pydantic model | Output berupa structured data |

## 4пёҸвғЈ LCEL Chain (LangChain Expression Language)

### Apa itu LCEL?

LCEL adalah **cara modern** untuk menyambungkan komponen LangChain menggunakan **pipe operator `|`**. Setiap komponen menerima output dari komponen sebelumnya.

### Konsep Dasar: Pipe `|`

```
Komponen_A | Komponen_B | Komponen_C
```

Artinya:
1. Jalankan `Komponen_A`, ambil hasilnya
2. Berikan hasil tersebut sebagai input ke `Komponen_B`
3. Berikan hasil dari `Komponen_B` ke `Komponen_C`

### Chain Sederhana: Model + Parser

```python
from langchain_core.output_parsers import StrOutputParser

parser = StrOutputParser()

# Sambungkan model dan parser dengan pipe |
chain = model | parser

# Invoke chain langsung dengan string
result = chain.invoke('how to master langchain and langgraph')
# result sudah berupa string, bukan AIMessage
```

**Alur data:**
```
"how to master..."  вҶ’  [Model]  вҶ’  AIMessage(...)  вҶ’  [Parser]  вҶ’  "string result"
```

### Chain Lengkap: Prompt + Model + Parser

```python
# Buat semua komponen
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are cultural young Indonesian, please translate teks into bahasa gaul di {kota}"),
    ("user", "{text}")
])
model_groq = ChatGroq(model='llama-3.3-70b-versatile')
parser = StrOutputParser()

# Sambungkan ketiganya menjadi satu chain
chainwithtemplate = prompt | model_groq | parser

# Invoke dengan dictionary (karena prompt butuh variabel)
result = chainwithtemplate.invoke({
    "kota": "Jakarta Selatan",
    "text": "Hello, please introduce my name is Aksa. I live in Jaksel"
})

print(result)
# Output: 'Halo, gue Aksa, tinggal di Jaksel, bro!'
```

**Alur data lengkap:**
```
{"kota": "Jakarta Selatan", "text": "Hello..."}
    вҶ“
[Prompt Template]  вҶ’  ChatPromptValue (list of messages)
    вҶ“
[Model (ChatGroq)]  вҶ’  AIMessage(content="Halo, gue Aksa...")
    вҶ“
[StrOutputParser]  вҶ’  "Halo, gue Aksa, tinggal di Jaksel, bro!"
```

## рҹ“Һ Tips & Best Practices

1. **Selalu gunakan `from_messages()`** untuk buat template, bukan `format_messages()`
2. **Gunakan `StrOutputParser`** di ujung chain supaya output siap pakai sebagai string
3. **Variabel di template** harus cocok dengan key di dictionary saat `.invoke()`
4. **LCEL chain bisa di-extend** вҖ” tambahkan komponen lain dengan menambah `| komponen` di belakang
5. **Gunakan model yang sesuai** вҖ” Groq untuk speed, Gemini untuk kemampuan multimodal
