## 3. Step 2: Text Splitting (Chunking)

### Mengapa Perlu Splitting?

```
PDF halaman 1 = 3000 karakter
PDF halaman 2 = 5000 karakter
                ─────────────
Total          = 8000 karakter

LLM context window = 8192 tokens ≈ 32,000 karakter
Embedding model    = 512 tokens  ≈ 2,000 karakter  ← Ini yang jadi limit!

Jika 1 chunk > 2000 karakter → embedding kurang akurat
Solusi → pecah jadi chunks 500-1000 karakter
```

### RecursiveCharacterTextSplitter (Recommended ✅)

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,       # Maksimal 1000 karakter per chunk
    chunk_overlap=200,     # Overlap 200 karakter antar chunk
    length_function=len,   # Fungsi untuk mengukur panjang
    separators=[           # Prioritas pemisah (default)
        "\n\n",            # 1. Paragraf baru
        "\n",              # 2. Baris baru
        " ",               # 3. Spasi
        ""                 # 4. Karakter (last resort)
    ]
)

chunks = text_splitter.split_documents(documents)
print(f"Dari {len(documents)} halaman → {len(chunks)} chunks")
```

### Visualisasi Overlap

```
Dokumen asli:
┌─────────────────────────────────────────────────────────────┐
│ Paragraf 1 xxxxxxxx │ Paragraf 2 xxxxxxxx │ Paragraf 3 xxx  │
└─────────────────────────────────────────────────────────────┘

Setelah split (chunk_size=1000, overlap=200):

Chunk 1: ┌─────────────────────┐
          │ Paragraf 1 xxxxxxxx │
          │ Paragraf 2 xx       │   ← 200 char overlap
          └─────────────────────┘
                          ▲
Chunk 2:            ┌─────┴───────────────┐
                    │ 2 xxxxxxxx          │
                    │ Paragraf 3 xxx      │   ← overlap lagi
                    └─────────────────────┘

Overlap menjaga agar konteks TIDAK terputus di boundary!
```

### Chunking Strategy Comparison

| Strategy | chunk_size | chunk_overlap | Best For |
|:---|:---|:---|:---|
| **Fine-grained** | 500 | 100 | FAQ, definisi singkat |
| **Balanced** ✅ | 1000 | 200 | General purpose, articles |
| **Coarse** | 2000 | 400 | Long-form research papers |
| **Semantic** | varies | varies | Needs `SemanticChunker` |

## 5. Step 4: Store ke PGVector

### Install LangChain PGVector

```bash
pip install langchain-postgres psycopg[binary]
```

### Store Documents

```python
from langchain_postgres import PGVector
from langchain_google_genai import GoogleGenerativeAIEmbeddings
# atau: from langchain_openai import OpenAIEmbeddings
# atau: from langchain_ollama import OllamaEmbeddings

# === Embedding Model ===
embeddings = GoogleGenerativeAIEmbeddings(model="models/text-embedding-004")

# === Connection String ===
CONNECTION_STRING = "postgresql+psycopg://langchain:langchain123@localhost:5432/vectorstore"

# === Method 1: Create from Documents (pertama kali) ===
vectorstore = PGVector.from_documents(
    documents=chunks,               # hasil text splitting
    embedding=embeddings,
    connection=CONNECTION_STRING,
    collection_name="pdf_documents", # nama "tabel" di DB
    pre_delete_collection=False,     # True = hapus data lama dulu
)

print(f"✅ {len(chunks)} chunks stored in PostgreSQL!")
```

### Connect ke Collection yang Sudah Ada

```python
# === Method 2: Connect ke data yang sudah ada ===
vectorstore = PGVector(
    connection=CONNECTION_STRING,
    collection_name="pdf_documents",
    embeddings=embeddings,           # Note: parameter name 'embeddings' (with s)
)

# Pakai seperti biasa
results = vectorstore.similarity_search("apa isi laporan?")
```

### Add More Documents Later

```python

new_loader = PyPDFLoader("./documents/laporan_baru.pdf")
```

## 6. Step 5: Query & Retrieval

### Similarity Search (Dasar)

```python
# Cari 4 dokumen paling mirip
query = "Apa pendapatan perusahaan tahun 2024?"
results = vectorstore.similarity_search(query, k=4)

for i, doc in enumerate(results):
    print(f"\n--- Result {i+1} ---")
    print(f"Source: {doc.metadata['source']}, Page: {doc.metadata['page']}")
    print(f"Content: {doc.page_content[:200]}...")
```

### Similarity Search with Score

```python
# Cari dengan skor relevansi (0 = paling mirip)
results = vectorstore.similarity_search_with_score(query, k=4)

for doc, score in results:
    print(f"Score: {score:.4f} | {doc.page_content[:100]}...")
```

### Sebagai Retriever (untuk Chain/RAG)

```python
# Convert ke Retriever — bisa langsung dipakai di chain
retriever = vectorstore.as_retriever(
    search_type="similarity",       # atau "mmr" (lebih diverse)
    search_kwargs={
        "k": 5,                     # ambil 5 dokumen
        # "score_threshold": 0.7,   # opsional: minimum relevansi
    }
)

# Pakai dalam RAG chain
from langchain.chains import create_retrieval_chain
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("""
Answer the question based only on the provided context:
<context>
{context}
</context>

Question: {input}
""")

from langchain_groq import ChatGroq
llm = ChatGroq(model="llama-3.3-70b-versatile")

document_chain = create_stuff_documents_chain(llm, prompt)
retrieval_chain = create_retrieval_chain(retriever, document_chain)

response = retrieval_chain.invoke({"input": "Apa pendapatan tahun 2024?"})
print(response["answer"])
```

## 7. Chunking Best Practices

### Rule of Thumb

```
chunk_size = 80% dari embedding model max tokens

Contoh:
- text-embedding-004 (Google)  → max 2048 tokens → chunk_size ≈ 1500 chars
- text-embedding-ada-002 (OpenAI) → max 8191 tokens → chunk_size ≈ 1000-2000 chars
- nomic-embed-text (Ollama)    → max 8192 tokens → chunk_size ≈ 1000-2000 chars

Rekomendasi universal: chunk_size = 1000, chunk_overlap = 200
```

### Kenapa Overlap Penting?

```
Tanpa overlap (chunk_overlap=0):
  Chunk 1: "Perusahaan ABC didirikan tahun 2010 oleh"
  Chunk 2: "Budi Santoso. Revenue tahun 2024 adalah..."
  
  Query: "Siapa pendiri perusahaan ABC?"
  → Chunk 1 match, tapi jawaban terpotong! ❌

Dengan overlap (chunk_overlap=200):
  Chunk 1: "Perusahaan ABC didirikan tahun 2010 oleh Budi Santoso. Revenue..."
  Chunk 2: "...oleh Budi Santoso. Revenue tahun 2024 adalah 50 miliar..."
  
  Query: "Siapa pendiri perusahaan ABC?"
  → Chunk 1 match, jawaban lengkap! ✅
```

### Advanced: Semantic Chunking

```python
# Memecah berdasarkan MAKNA, bukan jumlah karakter
from langchain_experimental.text_splitter import SemanticChunker

semantic_splitter = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="percentile",  # atau "standard_deviation"
)

semantic_chunks = semantic_splitter.split_documents(documents)
# Setiap chunk berisi satu "ide" yang utuh
```

## 8. Complete Production Code using LCEL

> **LCEL (LangChain Expression Language)** adalah cara **modern dan recommended** untuk membangun chain di LangChain v0.2+. Versi ini sudah **dilengkapi chat history** sehingga user bisa bertanya lanjutan (follow-up) tanpa kehilangan konteks percakapan.

### 🧠 Masalah: RAG Tanpa History = Buta Konteks

```
❌ RAG Stateless (tanpa chat history):

User: "Apa isi dokumen ini?"        → AI: "Dokumen ini berisi laporan keuangan 2024..."
User: "Berapa pendapatannya?"       → AI: "Saya tidak tahu pendapatan APA yang dimaksud." 😕
                                         ↑ Retriever cari "pendapatannya" → tidak ketemu!

✅ RAG dengan Chat History:

User: "Apa isi dokumen ini?"        → AI: "Dokumen ini berisi laporan keuangan 2024..."
User: "Berapa pendapatannya?"       → AI: "Pendapatan perusahaan tahun 2024 adalah 50 miliar." ✅
     ↑ System otomatis rewrite     → "Berapa pendapatan di laporan keuangan 2024?"
       lalu cari di retriever          ↑ Retriever match! 🎯
```

### 🏗️ Arsitektur: 2 Chain yang Bekerja Bersama

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     CONVERSATIONAL RAG (LCEL)                          │
│                                                                         │
│  ┌──────────────────────────┐                                           │
│  │  CHAIN 1: Contextualize  │  "Berapa pendapatannya?"                  │
│  │  (Rewrite Question)      │  + chat_history                           │
│  │                          │  → "Berapa pendapatan di laporan 2024?"   │
│  └────────────┬─────────────┘                                           │
│               │                                                         │
│               ▼                                                         │
│  ┌──────────────────────────┐                                           │
│  │  CHAIN 2: RAG Answer     │  question + context dari retriever        │
│  │  (Retrieve + Generate)   │  → "Pendapatan tahun 2024 adalah..."      │
│  └──────────────────────────┘                                           │
│                                                                         │
│  Dibungkus oleh: RunnableWithMessageHistory (auto save/load session)    │
└─────────────────────────────────────────────────────────────────────────┘
```

### Full Production Code (LCEL + Chat History)

```python
"""
PDF → PGVector → Conversational RAG Pipeline (Production-Ready, LCEL)
Dengan Chat History — user bisa bertanya lanjutan (follow-up questions)
"""

import os
from dotenv import load_dotenv
from operator import itemgetter
load_dotenv()

# ============================================
# 1. LOAD PDF
# ============================================
from langchain_community.document_loaders import PyPDFDirectoryLoader

loader = PyPDFDirectoryLoader("./documents/")
documents = loader.load()
print(f"📄 Loaded {len(documents)} pages from PDF files")

# ============================================
# 2. CHUNK
# ============================================
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200,
)
chunks = text_splitter.split_documents(documents)
print(f"✂️ Split into {len(chunks)} chunks")

# ============================================
# 3. EMBEDDING MODEL
# ============================================
from langchain_google_genai import GoogleGenerativeAIEmbeddings

embeddings = GoogleGenerativeAIEmbeddings(
    model="models/text-embedding-004"
)

# ============================================
# 4. STORE TO PGVECTOR
# ============================================
from langchain_postgres import PGVector

CONNECTION = "postgresql+psycopg://langchain:langchain123@localhost:5432/vectorstore"

vectorstore = PGVector.from_documents(
    documents=chunks,
    embedding=embeddings,
    connection=CONNECTION,
    collection_name="my_pdf_knowledge",
    pre_delete_collection=False,
)
print(f"✅ Stored {len(chunks)} chunks in PGVector!")

# ============================================
# 5. SETUP RETRIEVER
# ============================================
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 5}
)

# ============================================
# 6. KOMPONEN DASAR
# ============================================
from langchain_groq import ChatGroq
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough, RunnableBranch, RunnableLambda
from langchain_core.messages import HumanMessage, AIMessage, trim_messages

llm = ChatGroq(model="llama-3.3-70b-versatile")

def format_docs(docs):
    """Gabungkan page_content semua dokumen jadi satu string."""
    return "\n\n".join(doc.page_content for doc in docs)

# ============================================
# 7. CHAIN 1: CONTEXTUALIZE QUESTION
#    (Rewrite pertanyaan berdasarkan history)
# ============================================
contextualize_q_prompt = ChatPromptTemplate.from_messages([
    ("system",
     "Given a chat history and the latest user question "
     "which might reference context in the chat history, "
     "formulate a standalone question which can be understood "
     "WITHOUT the chat history. Do NOT answer the question, "
     "just reformulate it if needed and otherwise return it as is."),
    MessagesPlaceholder(variable_name="chat_history"),
    ("human", "{input}"),
])

# Chain: jika ada history → rewrite, jika tidak → langsung teruskan
contextualize_q_chain = contextualize_q_prompt | llm | StrOutputParser()

# ============================================
# 8. CHAIN 2: RAG (Retrieve + Answer)
# ============================================
qa_prompt = ChatPromptTemplate.from_messages([
    ("system",
     "You are an assistant for question-answering tasks. "
     "Use the following pieces of retrieved context to answer "
     "the question. If you don't know the answer, say that you "
     "don't know. Use three sentences maximum and keep the "
     "answer concise."),
    MessagesPlaceholder(variable_name="chat_history"),
    ("human", "{input}"),
])

# --- Fungsi: Apakah ada chat history? ---
def contextualized_question(input_dict):
    """Jika ada chat history, rewrite pertanyaan dulu. Jika tidak, langsung pakai."""
    if input_dict.get("chat_history"):
        return contextualize_q_chain
    else:
        return input_dict["input"]

# --- Susun RAG Chain (LCEL) ---
rag_chain = (
    RunnablePassthrough.assign(
        context=contextualized_question | retriever | format_docs
    )
    | qa_prompt
    | llm
    | StrOutputParser()
)

# ============================================
# 9. WRAP DENGAN MESSAGE HISTORY
# ============================================
from langchain_community.chat_message_histories import ChatMessageHistory
# Untuk production, ganti dengan:
# from langchain_community.chat_message_histories import SQLChatMessageHistory
# from langchain_postgres import PostgresChatMessageHistory

from langchain_core.chat_history import BaseChatMessageHistory
from langchain_core.runnables.history import RunnableWithMessageHistory

store = {}  # In-memory. Ganti dengan DB untuk production!

def get_session_history(session_id: str) -> BaseChatMessageHistory:
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]

conversational_rag = RunnableWithMessageHistory(
    rag_chain,
    get_session_history,
    input_messages_key="input",
    history_messages_key="chat_history",
)

# ============================================
# 10. TANYA! (Conversational)
# ============================================
config = {"configurable": {"session_id": "user-001"}}

# Pertanyaan 1
response = conversational_rag.invoke(
    {"input": "Apa isi dokumen ini?"},
    config=config,
)
print(f"\n🤖 Answer 1: {response}")

# Pertanyaan 2 (follow-up — pakai konteks dari pertanyaan 1!)
response = conversational_rag.invoke(
    {"input": "Bisa jelaskan lebih detail?"},
    config=config,
)
print(f"\n🤖 Answer 2: {response}")
```

### Penjelasan Komponen

| Komponen | Fungsi | Penjelasan |
|:---|:---|:---|
| `contextualize_q_prompt` | Template rewrite | Minta LLM ubah pertanyaan follow-up jadi standalone question |
| `contextualize_q_chain` | Rewrite chain | `prompt \| llm \| parser` — menghasilkan pertanyaan yang berdiri sendiri |
| `contextualized_question()` | Routing logic | Cek: ada history? → rewrite. Kosong? → langsung pakai input |
| `RunnablePassthrough.assign()` | Tambah key `context` | Menjalankan retriever dan menyimpan hasilnya sebagai `context` |
| `qa_prompt` | Template jawab | Berisi system prompt + chat_history (MessagesPlaceholder) + input |
| `RunnableWithMessageHistory` | Auto history | Otomatis load history sebelum invoke, save setelah invoke |
| `get_session_history()` | Session manager | Return `ChatMessageHistory` object berdasarkan session_id |

### Kenapa Perlu "Contextualize Question"?

```
Tanpa Contextualize:
  User: "Apa itu PGVector?"     → Retriever cari "PGVector" → ✅ Ketemu
  User: "Bagaimana instalasinya?"  → Retriever cari "instalasinya" → ❌ GAGAL
                                     ↑ Retriever tidak tahu "nya" = PGVector!

Dengan Contextualize:
  User: "Apa itu PGVector?"     → Retriever cari "PGVector" → ✅ Ketemu
  User: "Bagaimana instalasinya?"
         ↓ Rewrite oleh LLM
       "Bagaimana cara instalasi PGVector?"
         ↓
       Retriever cari "instalasi PGVector" → ✅ Ketemu!
```

### LCEL + History vs Legacy Helper: Perbandingan

| Aspek | Legacy (`create_retrieval_chain`) | LCEL + History |
|:---|:---|:---|
| **Chat History** | Perlu `create_history_aware_retriever` | Eksplisit, terlihat di kode |
| **Transparansi** | Logika tersembunyi di helper | Setiap step TERLIHAT JELAS |
| **Streaming** | Perlu setup khusus | `.stream()` langsung bekerja |
| **Trimming** | Harus manual inject | `RunnablePassthrough.assign()` |
| **Storage swap** | Harus refactor | Ganti `get_session_history()` saja |
| **Input key** | `{"input": "..."}` | `{"input": "..."}` (sama) |
| **Output** | `{"answer": "...", "context": [...]}` | String langsung |
| **Future-proof** | Ada risiko deprecated | Standar baru LangChain v0.2+ |

### Quick Reference: Perbedaan RAG Biasa vs Conversational RAG

```python
# === RAG Biasa (tanpa history) ===
rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt | llm | StrOutputParser()
)
response = rag_chain.invoke("Pertanyaan?")

# === Conversational RAG (dengan history) ===
rag_chain = (
    RunnablePassthrough.assign(
        context=contextualized_question | retriever | format_docs  # ← rewrite dulu!
    )
    | qa_prompt    # ← pakai MessagesPlaceholder untuk history
    | llm
    | StrOutputParser()
)
conversational_rag = RunnableWithMessageHistory(rag_chain, get_session_history, ...)
response = conversational_rag.invoke({"input": "Pertanyaan?"}, config=config)
```

## 9. Complete Production Code (Legacy Helper)

```python
"""
PDF → PGVector Pipeline (Production-Ready)
"""

import os
from dotenv import load_dotenv
load_dotenv()

# ============================================
# 1. LOAD PDF
# ============================================
from langchain_community.document_loaders import PyPDFDirectoryLoader

loader = PyPDFDirectoryLoader("./documents/")
documents = loader.load()
print(f"📄 Loaded {len(documents)} pages from PDF files")

# ============================================
# 2. CHUNK
# ============================================
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200,
)
chunks = text_splitter.split_documents(documents)
print(f"✂️ Split into {len(chunks)} chunks")

# ============================================
# 3. EMBEDDING MODEL
# ============================================
from langchain_google_genai import GoogleGenerativeAIEmbeddings

embeddings = GoogleGenerativeAIEmbeddings(
    model="models/text-embedding-004"
)

# ============================================
# 4. STORE TO PGVECTOR
# ============================================
from langchain_postgres import PGVector

CONNECTION = "postgresql+psycopg://langchain:langchain123@localhost:5432/vectorstore"

vectorstore = PGVector.from_documents(
    documents=chunks,
    embedding=embeddings,
    connection=CONNECTION,
    collection_name="my_pdf_knowledge",
    pre_delete_collection=False,
)
print(f"✅ Stored {len(chunks)} chunks in PGVector!")

# ============================================
# 5. QUERY (RAG)
# ============================================
from langchain_groq import ChatGroq
from langchain.chains import create_retrieval_chain
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain_core.prompts import ChatPromptTemplate

llm = ChatGroq(model="llama-3.3-70b-versatile")
retriever = vectorstore.as_retriever(search_kwargs={"k": 5})

prompt = ChatPromptTemplate.from_template("""
Answer based only on the context below. If unsure, say "I don't know."

<context>
{context}
</context>

Question: {input}
""")

document_chain = create_stuff_documents_chain(llm, prompt)
rag_chain = create_retrieval_chain(retriever, document_chain)

# Tanya!
response = rag_chain.invoke({"input": "Apa isi dokumen ini?"})
print(f"\n🤖 Answer: {response['answer']}")
```

## 10. Perbandingan VectorStore

| Feature | Chroma | FAISS | **PGVector** |
|:---|:---|:---|:---|
| **Storage** | File lokal | In-memory/file | PostgreSQL |
| **Setup** | Mudah | Mudah | Perlu DB server |
| **Scalability** | Juta docs | Juta docs | **Ratusan juta** |
| **Persistence** | ✅ File | ❌ Memory | ✅ **Database** |
| **Multi-user** | ❌ | ❌ | ✅ **ACID compliant** |
| **Backup** | Copy folder | Export file | **pg_dump, replication** |
| **Filtering** | Metadata filter | Basic | **SQL + metadata** |
| **Cloud options** | Self-host | Self-host | **Supabase, Neon, RDS** |
| **Best for** | Prototyping | Speed | **Production** ✅ |

### Decision Guide

```
Baru mulai / belajar?
├── Ya → Chroma (yang kamu pakai sekarang, sudah bagus!)
│
└── Mau deploy ke production?
    ├── Single user + speed penting? → FAISS
    ├── Multi-user + persistence?    → PGVector ✅
    └── Serverless + scaling?        → Pinecone / Weaviate (managed)
```
